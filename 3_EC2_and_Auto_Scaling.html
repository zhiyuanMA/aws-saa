EC2 & Auto Scaling

194 道题 · 7 个子知识点

核心知识点

  • Spot：最便宜（省90%），可被中断，仅适合容错无状态工作负载
  • Reserved Instance：1/3年承诺，省72%；Convertible可换实例家族，Standard不可
  • Savings Plans：比RI更灵活的承诺，覆盖EC2/Lambda/Fargate
  • Auto Scaling策略：Simple/Step/Target Tracking（推荐）/Scheduled/Predictive
  • gp3：最高16,000 IOPS，可独立配置IOPS和吞吐量；io2：最高64,000 IOPS
  • st1：大数据顺序读写HDD；sc1：最便宜冷存储HDD
  • Instance Store：物理附加，IOPS极高，实例停止/终止后数据永久丢失
  • Cluster置放群组：同AZ低延迟HPC；Spread：不同机架容错，每AZ最多7实例；Partition：分布式大数据
  • Launch Template：支持多版本和混合Spot/On-Demand舰队，优于Launch Configuration
  • SSM Session Manager：无需SSH/Bastion即可安全访问EC2

解题关键词模式

  • 最低成本 + 可中断 → Spot Instance
  • 稳定长期负载 → Reserved Instance或Savings Plans
  • 低延迟HPC紧密耦合 → Cluster Placement Group
  • IOPS > 16,000 → io2/io1
  • 大数据顺序读写 → st1
  • 批量补丁/命令 → SSM Patch Manager/Run Command
  • 无需SSH安全访问 → SSM Session Manager

⚠️ 常见陷阱

  • Reserved Convertible可换实例家族；Standard不能换
  • gp2 IOPS随容量线性增长(3 IOPS/GB)；gp3可独立配置，基础价格更低
  • Launch Template支持Spot+On-Demand混合；Launch Configuration不支持
  • Auto Scaling cooldown期间不触发新扩展；Scale-in保护防误删
