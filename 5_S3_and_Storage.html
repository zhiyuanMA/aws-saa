S3 & Storage

113 道题 · 5 个子知识点

核心知识点

  • 存储类成本排序（低→高）：Deep Archive < Glacier Flexible < Glacier Instant < One Zone-IA < Standard-IA < Standard
  • Intelligent-Tiering：自动分层，适合不可预测访问模式，无检索费
  • Transfer Acceleration：利用边缘节点加速全球上传，适合多地上传到单一桶
  • CRR（跨Region复制）：需启用版本控制，异步复制，用于合规/降低跨Region延迟
  • Object Lock Compliance：任何人（含root）不能删除，满足最严格合规要求
  • Presigned URL：带过期时间的临时访问链接，无需修改桶策略
  • S3 Select：SQL查询提取对象子集，减少传输量
  • 版本控制：启用后不能关闭，只能暂停(Suspend)
  • Multipart Upload：>100MB推荐，>5GB必须使用

解题关键词模式

  • 全球多地上传到单一S3桶 → Transfer Acceleration
  • 存档/合规/不常访问 → Glacier系列（按检索时间选）
  • 未知访问模式/自动分层 → Intelligent-Tiering
  • 跨Region合规复制 → CRR（需版本控制）
  • 防止对象删除/合规保留 → Object Lock Compliance模式
  • 临时访问私有对象 → Presigned URL
  • 查询CSV/JSON部分数据 → S3 Select

⚠️ 常见陷阱

  • One Zone-IA只有一个AZ，AZ故障会丢数据；适合可重新生成的数据
  • CRR只复制启用后的新对象，已有对象需用S3 Batch Operations
  • Glacier Instant：毫秒级检索；Flexible：分钟到小时；Deep Archive：12-48小时
  • Transfer Acceleration距离越近效果越不明显
