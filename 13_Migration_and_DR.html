Migration & DR

12 道题 · 2 个子知识点

核心知识点

  • Backup & Restore：最低成本，RTO/RPO小时级，备份到S3灾难时重建
  • Pilot Light：核心组件（数据库）持续运行，其他灾难时快速启动，RTO分钟到小时
  • Warm Standby：缩减规模的完整系统持续运行，灾难时扩展，RTO分钟级
  • Multi-Site Active/Active：两Region完全运行，近零RTO/RPO，成本最高
  • Application Migration Service(MGN)：Lift-and-Shift，持续复制，最小切割窗口
  • DMS：数据库迁移，支持同构/异构，持续CDC复制
  • AWS Backup：集中管理跨服务备份（EBS/RDS/DynamoDB/EFS/FSx/S3/EC2）

解题关键词模式

  • 最低成本DR/RTO可接受小时级 → Backup & Restore
  • 快速恢复/成本适中 → Pilot Light或Warm Standby
  • 近零RTO/RPO → Multi-Site Active/Active
  • Lift-and-Shift服务器迁移 → Application Migration Service
  • 数据库异构迁移(Oracle→Aurora) → DMS + SCT

⚠️ 常见陷阱

  • Pilot Light只运行核心（数据库），其他需启动；Warm Standby整个系统都在跑但规模缩减
  • RTO：恢复时间（多快恢复）；RPO：恢复点（最多丢多少数据）
