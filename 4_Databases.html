Databases

134 道题 · 7 个子知识点

核心知识点

  • RDS Multi-AZ：同步复制，自动故障转移，用于HA，不用于读扩展
  • Read Replica：异步复制，用于读扩展，可跨Region，需手动提升
  • Aurora：存储自动扩展(10GB→128TB)，6份副本跨3AZ，Read Replica可自动故障转移
  • Aurora Global Database：<1秒跨Region复制，次Region可提升为主
  • Aurora Serverless v2：按需自动扩展，适合不可预测负载
  • DynamoDB：毫秒级NoSQL；DAX：微秒级缓存，无需改代码
  • DynamoDB Global Tables：多Region多活(active-active)，自动双向复制
  • ElastiCache Redis：持久化+复制+数据结构；Memcached：纯缓存，多线程，无持久化
  • RDS Proxy：连接池，解决Lambda/无服务器大量短连接耗尽RDS连接的问题

解题关键词模式

  • 读密集/读扩展 → Read Replica或ElastiCache
  • 自动故障转移/HA → Multi-AZ或Aurora
  • 跨Region灾难恢复 → Aurora Global Database
  • 无服务器/不可预测负载 → Aurora Serverless v2或DynamoDB
  • 微秒级NoSQL → DynamoDB + DAX
  • Lambda + RDS连接超限 → RDS Proxy
  • 多Region多活NoSQL → DynamoDB Global Tables

⚠️ 常见陷阱

  • Read Replica ≠ 高可用；Multi-AZ ≠ 读扩展，用途不同
  • Aurora Read Replica可自动故障转移；RDS Read Replica需手动提升
  • ElastiCache Redis支持持久化和事务；Memcached不支持
  • RDS Proxy不减少延迟，只管理连接池减少连接压力
