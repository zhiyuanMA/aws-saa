Load Balancing

18 道题 · 1 个子知识点

核心知识点

  • ALB（Layer 7）：HTTP/HTTPS/WebSocket，路径/Host/Header路由，适合微服务
  • NLB（Layer 4）：TCP/UDP/TLS，超低延迟，支持静态Elastic IP
  • GLB（Layer 3）：透明插入第三方安全设备（防火墙/IDS），GENEVE协议
  • Cross-Zone：ALB默认开启且免费；NLB默认关闭，开启后收费
  • Sticky Session：ALB基于Cookie，NLB基于源IP
  • ALB没有静态IP（用DNS名）；NLB支持Elastic IP

解题关键词模式

  • HTTP路由/微服务/容器 → ALB
  • 高性能TCP/UDP/游戏/IoT → NLB
  • 第三方防火墙/安全设备 → GLB
  • 需要固定IP地址 → NLB（ALB没有固定IP）
  • WebSocket → ALB

⚠️ 常见陷阱

  • ALB没有静态IP；NLB有Elastic IP
  • GLB不用于普通应用负载均衡，是用来透明插入安全设备的
  • Cross-Zone：ALB默认启用且免费；NLB/GLB默认禁用且启用后收费
