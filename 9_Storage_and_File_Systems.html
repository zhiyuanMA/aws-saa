Storage & File Systems

34 道题 · 4 个子知识点

核心知识点

  • EFS：NFS协议，多EC2实例跨AZ共享挂载，仅限Linux
  • FSx for Windows：SMB协议，集成AD，Windows专用
  • FSx for Lustre：高性能并行文件系统，亚毫秒延迟，可原生集成S3，适合HPC/ML
  • FSx for NetApp ONTAP：多协议(NFS/SMB/iSCSI)，支持压缩/去重
  • Storage Gateway File Gateway：将S3呈现为本地NFS/SMB，混合云文件访问
  • Storage Gateway Volume Gateway：iSCSI块存储 + S3异步备份
  • Storage Gateway Tape Gateway：虚拟磁带库，替换物理磁带备份，数据存Glacier
  • DataSync：数据迁移/同步（本地→AWS），比手动脚本快10倍，全托管
  • Snowball Edge：80-210TB物理设备，网络不足时的大规模迁移
  • Transfer Family：SFTP/FTPS/FTP直接传输到S3/EFS，无需修改现有工作流

解题关键词模式

  • 多EC2实例共享文件/NFS/Linux → EFS
  • Windows文件共享/SMB/AD → FSx for Windows
  • HPC/ML/高性能文件系统 → FSx for Lustre
  • 本地到AWS数据迁移同步 → DataSync
  • 网络带宽不足大量数据迁移 → Snowball Edge
  • 混合云NFS/SMB访问S3 → Storage Gateway File Gateway
  • SFTP用户直接上传到S3 → Transfer Family

⚠️ 常见陷阱

  • EFS是Linux(POSIX)文件系统；Windows需用FSx for Windows
  • EBS只能挂载到单个EC2（multi-attach仅限io2且同AZ）；EFS可多实例多AZ共享
  • DataSync用于迁移；Storage Gateway用于持续混合云访问
