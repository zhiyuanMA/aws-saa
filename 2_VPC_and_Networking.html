VPC & Networking

118 道题 · 6 个子知识点

核心知识点

  • 安全组：有状态，实例级，仅Allow规则，返回流量自动允许
  • NACL：无状态，子网级，有编号规则，需显式配置出入双向，支持Deny
  • NAT Gateway：私有子网访问互联网，每AZ部署一个避免跨AZ费用
  • VPC Peering：无传递性；Transit Gateway：支持传递路由，Hub-Spoke架构
  • Gateway型Endpoint：S3/DynamoDB免费；Interface型(PrivateLink)：其他服务，按小时收费
  • Direct Connect：专用物理线路，1-100Gbps，低延迟；不加密，需叠加IPSec VPN加密
  • Site-to-Site VPN：数小时部署，走公网加密隧道，适合快速连接或DC备份
  • Flow Logs：记录IP流量元数据（不含内容），存S3或CloudWatch
  • Egress-Only Internet Gateway：IPv6出站专用，相当于IPv6的NAT Gateway

解题关键词模式

  • 私网访问S3不走公网 → VPC Gateway Endpoint（免费）
  • 多VPC互联/Hub-Spoke → Transit Gateway
  • 本地数据中心高带宽低延迟 → Direct Connect
  • 快速建立本地到AWS连接 → Site-to-Site VPN
  • 阻止特定IP/CIDR → NACL（SG无法显式拒绝）
  • 跨账户私有访问服务 → PrivateLink
  • DC备份方案 → VPN作为Direct Connect的备份

⚠️ 常见陷阱

  • 安全组无法显式拒绝；需要黑名单用NACL
  • VPC Peering无传递性：需要大规模互联用Transit Gateway
  • NAT Gateway放在公有子网，私有子网路由指向它
  • Gateway型Endpoint只有S3和DynamoDB；其他服务用Interface型（收费）
  • Direct Connect不加密，需要IPSec VPN叠加实现加密传输
