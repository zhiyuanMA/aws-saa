Monitoring & Management

44 道题 · 4 个子知识点

核心知识点

  • CloudWatch Metrics：默认5分钟，详细监控1分钟，自定义指标支持1秒
  • CloudWatch Alarms：触发Auto Scaling/SNS/EC2操作；Composite Alarm组合多个条件
  • CloudWatch Logs：日志收集；Log Insights：SQL查询；Metric Filter：从日志提取指标
  • CloudTrail：记录所有API调用（谁/何时/做了什么），默认保留90天，更长需导出S3
  • AWS Config：记录配置变更历史，持续检测合规性，但不能阻止违规操作
  • SSM：Run Command/Patch Manager/Session Manager/Parameter Store/Automation
  • Trusted Advisor：5类最佳实践建议（成本/性能/安全/容错/配额）
  • Control Tower：自动化搭建多账户Landing Zone，内置guardrails

解题关键词模式

  • 告警触发Auto Scaling → CloudWatch Alarm
  • 谁做了什么操作/API审计 → CloudTrail
  • 资源配置是否合规/变更历史 → AWS Config
  • 无需SSH管理EC2 → SSM Session Manager
  • 批量打补丁 → SSM Patch Manager
  • 多账户统一治理 → Organizations + Control Tower

⚠️ 常见陷阱

  • CloudTrail记录API调用，不是应用日志；应用日志用CloudWatch Logs
  • AWS Config检测合规性但不能阻止违规；要阻止用SCP或IAM
