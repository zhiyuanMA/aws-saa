IAM & Security

161 道题 · 7 个子知识点

核心知识点

  • IAM Policy评估：显式Deny > Allow，默认Deny所有
  • SCP限制成员账户最大权限，但不授予权限，管理账户不受SCP约束
  • aws:PrincipalOrgID：限制只有组织内账户可访问资源
  • IAM Role + STS AssumeRole：跨账户访问标准模式，使用临时凭证
  • Cognito User Pool：应用用户认证（登录/注册）；Identity Pool：联合身份获取AWS临时凭证
  • KMS信封加密：CMK加密数据密钥，数据密钥加密数据，CMK不离开KMS
  • Secrets Manager：自动轮换密钥；Parameter Store：简单配置，成本更低但不支持自动轮换
  • GuardDuty：分析VPC Flow Logs/CloudTrail/DNS，ML检测威胁，无需Agent
  • Macie：发现S3中的PII/敏感数据；Inspector：EC2/容器漏洞扫描
  • WAF防SQL注入/XSS；Shield Standard免费/Advanced付费DDoS防护

解题关键词模式

  • 最小权限 + 跨账户 → IAM Role + STS AssumeRole
  • 组织内所有账户限制 → SCP
  • 只有组织内账户访问S3 → aws:PrincipalOrgID条件键
  • 应用用户登录 → Cognito User Pool
  • 第三方/SAML联合身份 → Cognito Identity Pool
  • 自动轮换数据库密码 → Secrets Manager
  • 检测可疑活动/威胁 → GuardDuty
  • 保护S3中PII数据 → Macie
  • DDoS防护 → Shield Advanced + CloudFront/Route 53
  • SQL注入/XSS防护 → WAF

⚠️ 常见陷阱

  • SCP不授予权限，即使SCP允许，还需IAM策略允许
  • 管理账户(Management Account)不受SCP限制
  • Parameter Store免费但不支持自动轮换；Secrets Manager按密钥收费但支持自动轮换
  • GuardDuty分析元数据日志；Macie分析S3内容本身
  • KMS不直接加密大量数据，使用信封加密（CMK→数据密钥→数据）
